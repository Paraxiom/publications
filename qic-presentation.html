Toward Provably Correct PQC

For post-quantum cryptography, "it works" is not enough.
The goal is code that is provably correct, panic-free, and secret-independent.

Kani (AWS)

Bounded model checking directly on Rust. Proves absence of panics, integer overflows, and undefined behavior across all inputs. Adding to CI pipeline.

Verus (MSR)

SMT-based proof obligations written in Rust syntax. Functional correctness verified by Z3 solver. Proof annotations live alongside implementation code.

Lean 4 + Mathlib

Mathematical foundations formalized in a proof assistant. Spectral gap theorems, Poincaré inequality on T². Aeneas bridges Rust ↔ Lean.

Reference Model: libcrux (Cryspen)

Cryspen's ML-KEM implementation uses hax to extract Rust into F*, prove correctness, panic freedom, and secret independence, then generate verified Rust. Their ML-KEM is the gold standard. We are building toward the same bar — same language, same ecosystem, same goal.
